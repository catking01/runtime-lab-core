# Architecture

Runtime Lab Core is organized around bounded, offline-verifiable runtime surfaces: deterministic helpers, admission, authority, descriptors, kernel boundary checks, offline LLM provider policy, read-only repository context, patch proposal, human-approved patch apply, allowlisted test running, and supervised agent-loop orchestration.

The public package intentionally excludes private evidence records, session logs, shell snapshots, credentials, private roadmap content, patent-sensitive material, and enterprise/private runtime surfaces.
